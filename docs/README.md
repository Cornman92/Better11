## Contributing to Documentation

Documentation contributions are highly valued! When contributing:

1. Use clear, concise language
2. Include screenshots and examples
3. Keep documentation up-to-date with code changes
4. Use proper markdown formatting
5. Test all code examples

## Documentation Standards

- Use Markdown format (.md)
- Follow the [Microsoft Writing Style Guide](https://docs.microsoft.com/en-us/style-guide/welcome/)
- Include code examples where helpful
- Add screenshots for UI-related documentation
- Keep documents organized and structured
